## 为什么要重构？

在2025年08月16日，我写下了我的第一篇博客，之后陆续更新了两篇内容，但随着使用，遇到了不少问题：

- **Hexo 架构老旧**
- **主题模板复杂**
- **没有现代前端能力**
- **加载速度慢**
- **...**

在此期间，朋友也给我推荐过其他主题，但我还是喜欢[anzhiyu主题](https://docs.anheyu.com/)这种的布局，也就没有更换。再加上事情比较多，所以很久都没有更新。几天前毕业论文完美收官，也有了几天空闲时间，最终决定用新框架重写整个博客。

> [!TIP]
>
> 在此感谢[小米mimo模型](https://platform.xiaomimimo.com/docs/zh-CN/welcome)送的200块钱Token😋，不然我也没这个动力与精力来重构

## 技术选型

经过对比，选择了以下方案：

| 层级     | 旧方案           | 新方案                     | 理由                                 |
| -------- | ---------------- | -------------------------- | ------------------------------------ |
| 框架     | Hexo 7.3         | **Astro 6**          | 内容优先的 SSG，零 JS 默认，构建极快 |
| 交互     | jQuery + 原生 JS | **React 19**         | 只在需要交互的组件使用（岛屿架构）   |
| 样式     | Stylus           | **Tailwind CSS 4**   | 实用优先，CSS-first 配置，开发效率高 |
| 模板     | Pug + EJS        | **Astro 组件**       | 更直观的组件化开发                   |
| 搜索     | Algolia          | **Pagefind**         | 零配置，构建时自动索引，无外部依赖   |
| 评论     | Waline           | **Waline（保留）**   | 已是独立服务，直接集成               |
| 代码高亮 | highlight.js     | **Shiki**            | Astro 内置，支持双主题，更准确       |
| 部署     | Vercel           | **Cloudflare Pages** | 全球 CDN                             |

## 设计系统

### 美化

#### 毛玻璃效果

以玻璃态设计语言：

```css
/* 所有卡片、菜单栏、下拉菜单统一使用 */
background: linear-gradient(135deg, rgba(255,255,255,0.4), rgba(255,255,255,0.1));
backdrop-filter: blur(20px);
border: 1px solid rgba(255,255,255,0.3);
border-radius: 16px;
box-shadow: 0 1px 24px rgba(0,0,0,0.1);
```

悬浮元素（下拉菜单）使用更高透明度补偿模糊导致的视觉透明：

```css
/* 二级菜单 */
background: linear-gradient(135deg, rgba(255,255,255,0.7), rgba(255,255,255,0.4));
```

#### 莫奈色系背景

亮色模式支持 8 种莫奈画作色系背景，在设置菜单中一键切换：

- 云端漫步（默认）、睡莲、日出印象、干草堆、紫藤、鲁昂大教堂、塞纳河、纯白

选择通过 `localStorage` 持久化，切换时背景色和边框色同步变化。

#### 深色模式星空

深色模式使用星空背景，通过 `body::before` 和 `body::after` 两层伪元素，用 `box-shadow` 绘制数百个 1px/2px 白色星点，配合 `animStar` 动画无限滚动。背景为深空渐变 `radial-gradient(ellipse at bottom, #1b2735 0%, #090a0f 100%)`。

### 动画系统

- `slide-in`：文章卡片入场（0.6s，交错延迟）
- `fade-in-up`：通用淡入上移
- `floating`：标签悬浮动画（6s 循环）
- `gradient-shift`：一言模块渐变背景（15s 循环）
- `settings-spin`：设置按钮图标旋转（2s 循环）
- `animStar`：星空滚动（50s/100s 双层）

## 首页设计

首页采用模块化布局，大部分布局还是保留[anzhiyu主题](https://docs.anheyu.com/)的样式，从上到下依次为：

### 1. 浮动胶囊导航栏

固定顶部的圆角胶囊形菜单栏，毛玻璃效果。左侧站点切换按钮 + Logo（hover 时图标滑入中心覆盖文字的动画），中间四个一级菜单（文章、友链、我的、关于）悬浮展开横向二级菜单，右侧设置按钮（uiverse.io 风格：白色胶囊，hover 变蓝 + 图标旋转）。

### 2. 即刻模块

双栏上下反向滚动的随笔轮播，数据来自 `essay.yml`。hover 时暂停滚动，单项 hover 变色 + 右移反馈。

### 3. 一言模块 + 分类卡片 + 随机图

一言模块显示 `hitokoto.yml` 中的随机语录，渐变动画背景，刷新按钮切换。下方三个分类卡片（日常、技术、工具）hover 发光。右侧随机图模块可切换为文章推荐列表。

### 4. 排序筛选 + 文章网格

React 组件实现的排序筛选栏：搜索框（模糊匹配标题/摘要/标签）+ 分类筛选 + 时间/浏览量升降序。下方双栏文章网格，每张卡片毛玻璃效果，封面图上方，标签/标题/时间/浏览数/未读标记在下方。

### 5. 侧边栏

个人简介卡片 + hello 动图（亮色/暗色自动切换） + 近期文章时间线。

## 文章详情页

文章详情页使用独立的毛玻璃卡片容器，分为标题卡片（封面图 + 标题 + 元信息 + 分类标签）和内容卡片（prose 排版 + TOC 目录侧边栏）。

目录支持 H2-H6 多级缩进显示，超过 8 个标题时自动折叠。点击目录链接会计算菜单栏高度偏移后平滑滚动。所有标题添加 `scroll-margin-top: 80px` 防止被固定菜单栏遮挡。

代码块采用 macOS 风格工具栏（红黄绿圆点 + 语言标签），支持行号、复制按钮、长代码折叠展开。使用 Shiki 双主题（ayu-light / ayu-dark）。

## 组件架构

采用 Astro 岛屿架构，按需加载 React 组件：

| 加载策略                | 组件                                                                                                       | 说明                     |
| ----------------------- | ---------------------------------------------------------------------------------------------------------- | ------------------------ |
| 静态（零 JS）           | Header, Footer, PostCard, TOC, EssayMarquee, HitokotoModule, CategoryCards, RandomImageModule, RecentPosts | 服务端渲染，无客户端 JS  |
| `client:only="react"` | SortFilterPosts                                                                                            | 纯客户端渲染，含状态管理 |
| `client:load`         | ThemeToggle, Comments, Search                                                                              | 立即加载的关键交互       |
| `client:visible`      | BangumiList, Gallery, PhotoAlbum, MusicPlayer                                                              | 滚动到可见区域才加载     |

## 数据管理

所有内容通过 Astro Content Collections 管理：

- 博客文章：`src/content/blog/*.md`（Markdown，Zod schema 校验）
- 结构化数据：`src/content/data/*.yml`（友链、装备、随笔、相册、一言等）
- JSON 数据：追番列表、TMDB 元数据、个人信息

YAML 数据通过 `js-yaml` 在构建时读取（`fs.readFileSync` + `yaml.load`），避免了 `@rollup/plugin-yaml` 的解析兼容问题。

## 部署

部署到 Cloudflare Pages，构建命令 `pnpm build`，输出 `dist/` 静态文件，全球 CDN 分发。

## 迁移过程

迁移分三个阶段：

1. **基础搭建**：Astro 项目初始化、Tailwind + shadcn/ui 配置、Content Collections、核心布局和页面
2. **功能完善**：自定义页面、搜索（Pagefind）、评论（Waline）、暗色模式、代码高亮等
3. **交互组件**：追番页、音乐播放器、相册、友链、留言板、每日一图等

Markdown 文章基本兼容，主要调整是 frontmatter 格式和 Hexo 特有标签（`{% post_link %}`、`{% asset_img %}`）的替换。

## 总结

这次重构把博客升级到了"现代、可维护、高性能"的状态。Astro 的岛屿架构让我可以在需要的地方用 React，而大部分页面保持零 JS。Tailwind CSS 4 让样式开发效率大幅提升，毛玻璃 + 莫奈色系 + 星空暗色模式的设计系统也比原主题更加统一和精致。

如果你也想搭建类似的博客，可以参考本项目的 [README.md](https://github.com/zzz2929/zzz-blog) 获取完整的自定义指南。
