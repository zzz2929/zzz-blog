博客最初部署在 [Vercel](https://vercel.com/) 上，但是：

1. **其他项目部署 Cloudflare**：两个平台来回切换太麻烦
2. **域名已经在 Cloudflare**：域名 DNS 托管已经放在 Cloudflare 了，既然都在一个平台，不如部署也放过去
3. **Cloudflare 有 D1数据库 + R2存储桶等**：可以为将来有可能的转动态博客做准备

## 迁移前的准备

### 确认当前状态

```
域名：blog.904002.xyz
DNS 托管：Cloudflare
部署平台：Vercel
框架：Hexo
构建命令：pnpm build
```

### 备份

迁移前先确认代码已经推送到 Git 仓库，确保随时可以回滚。

## Cloudflare Workers 配置

登录 [Cloudflare Dashboard](https://dash.cloudflare.com/) → Workers & Pages → Create → Workers → Connect to Git。

选择 GitHub 仓库，配置构建参数：

| 配置项                 | 值             |
| ---------------------- | -------------- |
| Framework preset       | Astro          |
| Build command          | `pnpm build` |
| Build output directory | `dist`       |
| Node.js version        | 22             |

## 添加域名

### 1. 添加自定义域名

在 Cloudflare Workers 项目 → Custom domains → Add custom domain，输入 `blog.904002.xyz`。

Cloudflare 会自动配置 DNS 记录。

### 2. 删除 Vercel 域名配置

在 Vercel 项目设置中移除 `blog.904002.xyz` 域名，避免冲突。

### 3. 等待 DNS 生效

DNS 切换通常在几分钟内生效，最长不超过 48 小时。
